# 護理排班系統 — Vibe Coding 注意事項

> 這份文件整理了開發過程中踩過的坑，幫助你在用 AI 產生程式碼時避開常見問題。
> 不需要懂程式，只要知道「該跟 AI 說什麼」就好。

---

## 一、什麼是 Vibe Coding？

簡單說就是：**你用口語描述你想要什麼功能，讓 AI 幫你寫程式。**

你不需要會寫程式，但需要知道一些「眉角」，才能讓 AI 寫出能用的東西。
以下就是我們實際碰到的問題和解法。

---

## 二、我們遇到的問題

### 問題 1：網頁打開後顯示「無法讀取試算表」

**發生原因（白話版）：**

我們的網頁是直接在電腦上雙擊打開的（像打開 Word 文件一樣），
但 Google 有安全規定：「從自己電腦直接打開的網頁，不能隨便跟 Google 伺服器要資料」。

這就像你從家裡打電話給銀行，銀行說：「這個號碼我不認識，不給你查帳。」

**解法：**

需要用一種叫「JSONP」的特殊方式來跟 Google 溝通，
這種方式可以繞過 Google 的安全限制。

**給 AI 的參考 Prompt：**

```
我要做一個 HTML 網頁，直接在電腦上雙擊打開（用 file:// 協定），
需要讀寫 Google Sheets 的資料。

重要：因為是用 file:// 開啟，fetch() 會被瀏覽器的 CORS 擋住，
所以前後端都必須使用 JSONP 模式：
- 前端用動態插入 <script> 標籤的方式呼叫
- Google Apps Script 後端回傳格式要用 callback(JSON) 包裝，
  MimeType 用 JAVASCRIPT（不是 JSON）
- 所有操作（讀取和寫入）都要走 doGet，不要用 doPost

請不要使用 fetch()、XMLHttpRequest 或 axios。
```

### 問題 2：試算表 ID 搞錯了，連到別人的表

**發生原因（白話版）：**

每個 Google 試算表都有一組專屬的 ID（就像身分證號碼），
AI 產生程式碼時可能會用到之前對話中提到的舊 ID，
或是自己編一個假的 ID。

**如何找到你的試算表 ID：**

打開你的 Google 試算表，看網址列：

```
https://docs.google.com/spreadsheets/d/這一段就是ID/edit#gid=0
                                       ^^^^^^^^^^^
```

`/d/` 和 `/edit` 之間的那一串文字就是你的試算表 ID。

**給 AI 的參考 Prompt：**

```
我的 Google 試算表網址是：
https://docs.google.com/spreadsheets/d/XXXXXX/edit#gid=0

請把 Apps Script 中的 SPREADSHEET_ID 設定為這個網址中的 ID。
```

### 問題 3：儲存按下去說成功，但試算表沒資料

**發生原因（白話版）：**

把資料傳給 Google 時，資料會先經過「打包」處理。
如果 AI 寫的程式碼不小心把資料打包了兩次（像信封裝了兩層），
Google 那邊打開信封後看到的還是一個信封，不是信的內容，就會處理失敗。

但因為信確實「寄出去」了，網頁就以為成功了。

**給 AI 的參考 Prompt：**

```
注意：使用 JSONP 傳送資料時，
jsonp 函式內部已經會對每個參數做 URL 編碼（encodeURIComponent），
所以呼叫 jsonp 時「不要」事先對參數做 encodeURIComponent，
否則會變成重複編碼，導致後端解析失敗。

同樣的，Google Apps Script 的 e.parameter 已經是自動解碼後的值，
後端不需要再做 decodeURIComponent。
```

---

### 問題 4：試算表是空的，但網頁顯示錯誤訊息

**發生原因（白話版）：**

第一次使用時，試算表本來就是空的，讀不到資料是正常的。
但 AI 寫的程式把「空的」和「出錯了」搞混了，
一律顯示錯誤訊息，讓人誤以為連線有問題。

**給 AI 的參考 Prompt：**

```
從 Google Sheets 讀取資料時，請區分三種情況：
1. 連線成功且有資料 → 正常載入
2. 連線成功但資料是空的 → 顯示友善提示「試算表是空的，已載入預設資料」
3. 連線失敗 → 才顯示錯誤訊息

不要把「空的試算表」當作錯誤處理。
```

## 三、完整的初始 Prompt 範本

以下是一個比較完整的 prompt，可以在請 AI 建立新的 Google Sheets 專案時使用：

```
請幫我做一個護理排班系統，包含一個 HTML 網頁和一個 Google Apps Script 後端。

【使用方式】
- HTML 檔案會在電腦上直接雙擊打開（file:// 協定），不會放到網站伺服器上
- 後端使用 Google Apps Script 部署為網頁應用程式

【技術限制 — 很重要，請務必遵守】
1. 因為 HTML 是用 file:// 開啟的，瀏覽器會擋 CORS，
   所以「不能用 fetch、XMLHttpRequest、axios」
2. 前後端通訊必須全部用 JSONP 模式：
   - 前端：用動態建立 <script> 標籤的方式發送請求
   - 後端：doGet 接收所有請求（讀取和寫入都走 GET）
   - 回傳格式：callback(JSON)，MimeType 用 JAVASCRIPT
3. 不要使用 doPost（從 file:// 無法正常發送 POST）
4. JSONP 函式內部會自動做 URL 編碼，呼叫時不要重複編碼
5. 後端的 e.parameter 已經自動解碼，不要再 decodeURIComponent

【Google 試算表設定】
- 試算表 ID：（在這裡貼上你的 ID）
- 工作表名稱：nurse
- 如果工作表不存在，程式要自動建立

【讀取資料的錯誤處理】
請區分「連線成功但資料為空」和「連線失敗」兩種情況，
空的試算表不要顯示錯誤訊息。

【部署設定】
- 前端：HTML 中要有設定區域讓使用者填入 Apps Script 的網址
- 網址要儲存到 localStorage，下次打開自動帶入
- 後端部署為「網頁應用程式」，執行身分=自己，存取權=所有人
```

## 四、部署 Google Apps Script 的步驟

每次修改 Apps Script 的程式碼後，都需要重新部署才會生效：

1. 打開你的 Google 試算表
2. 點選上方的「擴充功能」→「Apps Script」
3. 把新的程式碼貼進去（取代原本的內容）
4. 點選右上方的「部署」→「管理部署作業」
5. 點「編輯」（鉛筆圖示）
6. 「版本」欄位選「**新版本**」（很重要！不選新版本等於沒更新）
7. 按「部署」
8. 複製部署後的網址，貼到 HTML 的設定裡

> **常見錯誤**：改完程式碼後忘了選「新版本」就按部署，
> 結果還是跑舊的程式碼。每次改完都要選新版本！

## 五、快速檢查清單

遇到問題時，按照這個順序檢查：

- [ ] 試算表 ID 是否正確？（對照網址列確認）
- [ ] Apps Script 有沒有部署新版本？（最常被忘記的一步）
- [ ] HTML 設定裡的網址有沒有貼對？（要用 `/exec` 結尾的那個）
- [ ] 打開瀏覽器 F12 → Console 分頁看有沒有紅色錯誤訊息？
- [ ] 試算表是不是本來就是空的？（空的是正常的，存一次就有資料了）
